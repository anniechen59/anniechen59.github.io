Beyond Core Isolation: Non-Linear Memory Bandwidth Interference

Computer Architecture | Virtualization (WSL2) | Performance Characterization

Figure 1: The "Performance Cliff" identified during the 1KB to 64MB block size sweep.

The "Bandwidth Wall"

While physical CPU core pinning isolates pipeline contention, it fails to protect shared Last-Level Cache (LLC) and DRAM bandwidth. This study characterizes the non-linear performance collapse in multi-tenant workloads[cite: 280, 285].

Key Finding

49.9% Drop

Peak performance degradation observed at a 32MB block size[cite: 285].

Experimental Methodology

Conducted in a virtualized WSL2 environment (Ubuntu 22.04) on an Intel Core Ultra 5 125H (Meteor Lake) architecture[cite: 305, 314].

Black-Box Analysis

Since hardware PMUs are abstracted in WSL2, interference was inferred solely through application-level throughput and controlled sensitivity sweeps.

Isolation Strategy

Victim: sysbench memory (pinned to Core 0).
Aggressor: stress-ng-stream (pinned to Core 1)[cite: 323, 326, 329].

The "Shielding Effect"

Workloads within the 18MB LLC remained protected (latency-bound), while those exceeding it hit a "Bandwidth Wall"[cite: 310, 354, 371].

The 128KB Anomaly

A surprising 7.8% performance gain was observed at 128KB under interference. This is attributed to Intel Turbo Boost: the aggressor increased package power, boosting the clock frequency for the cache-resident victim[cite: 367, 368].

Effective LLC Boundary

Although the physical LLC is 18MB, severe interference began at 16MB (22.8% drop). This demonstrates the "Effective Capacity": OS kernel and metadata overhead reduce the usable buffer in shared environments[cite: 390, 391, 392].
